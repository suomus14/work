予実・課題管理について
障害対応において要件ごとに見積もりと実績工数を管理し、予実精度の向上に取り組んだ。従来は見積もり形式が曖昧だったが、テンプレート化により標準化と簡略化を実現。さらに過去の予実データを参照し、妥当性の高い工数算出へ改善した。
緊急対応における計画遵守について
委託元の緊急トラブル時にはサポートを実施。その際、別案件の進捗遅延が懸念されたため、事前に委託元へ相談し日程調整を行った。結果として全体計画への影響を回避し、計画通りの日程を遵守できた。
作業効率化の取り組み
リソースのグラフ化ツール導入やCoverityのGit連携により、業務効率化に貢献した。

CM活動や開発革新など、受託稼働に直接は係らない作業が多く発生した状況下でも、受託稼働率は75％以上を維持し、さらにCM活動を除いた場合には85％以上を達成することができた。
案件の谷間や作業待ちが生じた際には、提案型のアプローチで追加要件を受注し、稼働率向上に寄与した。
これらの取り組みにより、非受託業務との両立を図りつつ、課業目標を達成できた点は大きな成果であり、安定した稼働体制の確立に貢献できたと考える。
